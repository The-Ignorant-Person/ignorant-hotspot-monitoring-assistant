# 热点监控助手 - 需求分析

## 一、项目背景

作为计算机专业学生，需要第一时间获取技术热点（如 AI 大模型更新），不能依赖手动搜索。本项目通过自动化工具发现指定热点变化、发现最新热点，并及时发送通知。

项目定位：**轻量工具类项目**，采用敏捷开发方式。

## 二、目标用户

计算机专业学生/技术爱好者，希望：
- 追踪特定技术关键词（如 "GPT-5"、"Claude"）
- 自动发现技术领域内的热门话题
- 第一时间收到通知，不错过重要信息

## 三、核心功能

### 功能一：关键词监控 + 智能通知

**描述**：用户手动输入要监控的关键词，系统从多个信息源自动搜索，当关键词相关的真实内容出现时，第一时间发送通知。

**详细需求**：
1. 用户可以添加、删除、启用/禁用监控关键词
2. 每个关键词可设置作用域（如 "AI"、"编程语言"、"开源项目"）
3. 系统每 **30 分钟** 自动执行一轮扫描
4. 利用 AI（OpenRouter）识别内容真伪，过滤伪冒/标题党/无关内容
5. 发现真实热点后，通过以下方式通知用户：
   - **浏览器实时推送**（WebSocket + 浏览器 Notification API）
   - **邮件通知**（发送热点摘要邮件）

### 功能二：热点趋势自动收集

**描述**：每隔一段时间，自动搜集用户指定范围内的热点，汇总展示。

**详细需求**：
1. 用户输入关注范围（如 "AI 编程"、"大模型"）
2. 系统自动从多源聚合该范围内的热门内容
3. AI 分析热度、归类趋势、生成中文摘要
4. 以可视化方式展示趋势（热度等级、来源、时间线）

## 四、数据源（多源并行）

从多个信息源获取信息，避免单一信息源的局限：

| 数据源 | 获取方式 | 说明 |
|--------|----------|------|
| **网页搜索/爬虫** | Firecrawl SDK（`@mendable/firecrawl-js`） | 主力搜索源，通过网页搜索获取实时信息 |
| **Hacker News** | 公开 Firebase API | 技术社区热点，完全免费无需认证 |
| **RSS 订阅** | rss-parser 库解析 | TechCrunch、The Verge、Ben's Bites 等科技媒体 |
| **Twitter/X**（预留） | TwitterAPI.io REST API | 社交媒体热点（暂未接入，接口预留） |

## 五、通知方式

| 方式 | 场景 | 说明 |
|------|------|------|
| **浏览器实时推送** | 网页打开时 | WebSocket 推送 + Browser Notification API 弹窗 |
| **邮件通知** | 随时 | 每轮扫描发现新热点后，发送摘要邮件到用户配置的邮箱 |

## 六、AI 能力（OpenRouter）

通过 OpenRouter 对接 AI 大模型，实现：
1. **内容验证**：判断文章/消息是否为真实热点，识别标题党和伪冒内容
2. **相关性评分**：对内容与关键词的匹配程度打分（0-100）
3. **智能摘要**：生成中文摘要，方便快速了解热点内容
4. **趋势分析**：将多篇文章归类为趋势话题，评估热度等级

## 七、非功能需求

1. **前端页面独特**：不使用千篇一律的模板，采用"信号站/雷达监控站"主题风格
2. **轻量部署**：单机运行，无需复杂基础设施
3. **开发顺序**：先完成网页版并确保功能正常，再开发 Agent Skills
4. **扫描频率**：每 30 分钟自动扫描一轮
5. **数据持久化**：关键词、热点、配置等数据持久化存储

## 八、开发计划

| 阶段 | 内容 | 验收标准 |
|------|------|----------|
| 第一阶段 | 项目初始化 + UI 骨架 | 浏览器可看到完整的信号站风格界面 |
| 第二阶段 | 数据库 + 关键词管理 API | 关键词增删改查功能可用，数据持久化 |
| 第三阶段 | 多数据源接入 | 手动触发扫描可获取真实数据 |
| 第四阶段 | AI 集成（OpenRouter） | AI 能正确评分、过滤、生成摘要 |
| 第五阶段 | 定时监控 + 通知 | 30 分钟自动扫描 + 浏览器/邮件通知 |
| 第六阶段 | 打磨 + Agent Skills | 完整功能测试通过，Agent Skills 可用 |
