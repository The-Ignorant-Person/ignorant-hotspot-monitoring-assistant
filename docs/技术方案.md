# 热点监控助手 - 技术方案

## 一、技术栈选择

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | React + Vite + TailwindCSS | 快速开发、响应式、独特 UI |
| 后端 | Node.js + Express | 轻量、快速 |
| 数据存储 | SQLite（better-sqlite3） | 轻量级、无需额外配置、WAL 模式持久化可靠 |
| 热点数据源 | Firecrawl / Hacker News / RSS | 多源并行，避免单一依赖 |
| AI 识别 | OpenRouter API（OpenAI SDK） | 识别真假热点、内容分析、摘要生成 |
| 通知 | 邮件 / 浏览器推送 / WebSocket | 实时通知 |

## 三、数据库设计

使用 SQLite（better-sqlite3），WAL 模式保证并发读写性能。

```sql
-- 关键词表
CREATE TABLE keywords (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  keyword TEXT NOT NULL,
  scope TEXT DEFAULT 'AI',           -- 作用域：AI、编程语言、开源项目 等
  active INTEGER DEFAULT 1,          -- 是否启用
  created_at TEXT DEFAULT (datetime('now'))
);

-- 热点表
CREATE TABLE hotspots (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  source TEXT NOT NULL,              -- 来源：firecrawl / hackernews / rss / twitter
  title TEXT NOT NULL,               -- 标题
  summary TEXT,                      -- AI 生成的中文摘要
  url TEXT NOT NULL UNIQUE,          -- 原文链接（用于去重）
  score INTEGER DEFAULT 0,           -- AI 置信度 0-100
  verified INTEGER DEFAULT 0,        -- AI 是否验证通过
  reason TEXT,                       -- AI 判断理由
  keyword_id INTEGER,                -- 触发的关键词 ID
  published_at TEXT,                 -- 原文发布时间
  notified INTEGER DEFAULT 0,        -- 是否已发送通知
  created_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (keyword_id) REFERENCES keywords(id)
);

-- 趋势表
CREATE TABLE trends (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  category TEXT NOT NULL,            -- 分类
  title TEXT NOT NULL,               -- 趋势标题
  summary TEXT,                      -- 摘要
  heat_level INTEGER DEFAULT 1,      -- 热度 1-5
  sources TEXT,                      -- JSON 数组：相关文章 URL
  created_at TEXT DEFAULT (datetime('now'))
);

-- 设置表
CREATE TABLE settings (
  key TEXT PRIMARY KEY,              -- 配置键
  value TEXT NOT NULL                -- 配置值
);

-- 索引
CREATE INDEX idx_hotspots_url ON hotspots(url);
CREATE INDEX idx_hotspots_created ON hotspots(created_at);
CREATE INDEX idx_hotspots_keyword ON hotspots(keyword_id);
CREATE INDEX idx_hotspots_notified ON hotspots(notified);
```


### 4.4 监控

```
POST   /api/monitor/trigger       → 手动触发一轮扫描
GET    /api/status                → 系统状态（关键词数、热点数、上次扫描时间等）

```


## 五、数据源对接

### 5.1 Firecrawl — 网页搜索（主力源）

使用 `@mendable/firecrawl-js` SDK，项目已有 API Key。

```typescript
import Firecrawl from '@mendable/firecrawl-js';

const firecrawl = new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY });

// 搜索关键词相关的网页
const results = await firecrawl.search('AI大模型 最新发布', {
  limit: 5,
  scrapeOptions: { formats: ['markdown'] }
});

// 爬取特定页面内容
const doc = await firecrawl.scrape('https://example.com/article', {
  formats: ['markdown'],
  onlyMainContent: true,
  timeout: 30000
});
```

**注意**：控制请求频率，每轮扫描最多搜索 10 个关键词，每个关键词限制 5 条结果。

### 5.2 Hacker News — 技术社区（免费）

Firebase 公开 API，无需认证，无速率限制。

```typescript
// 获取热门故事 ID（最多 500 条）
const topIds = await fetch('https://hacker-news.firebaseio.com/v0/topstories.json')
  .then(r => r.json());

// 获取新故事
const newIds = await fetch('https://hacker-news.firebaseio.com/v0/newstories.json')
  .then(r => r.json());

// 获取单条故事详情
const story = await fetch(`https://hacker-news.firebaseio.com/v0/item/${id}.json`)
  .then(r => r.json());
// story: { id, title, url, score, time, by, descendants }
```

**策略**：每轮取 top 50 + new 50，按关键词过滤标题。

### 5.3 RSS 订阅 — 科技媒体

使用 `rss-parser` 库解析多个 RSS 源。

```typescript
import Parser from 'rss-parser';
const parser = new Parser();

const feeds = [
  'https://hnrss.org/frontpage?points=100',    // HN 高分
  'https://techcrunch.com/feed/',               // TechCrunch
  'https://www.theverge.com/rss/index.xml',     // The Verge
  'https://feeds.arstechnica.com/arstechnica/index',  // Ars Technica
];

for (const feedUrl of feeds) {
  try {
    const feed = await parser.parseURL(feedUrl);
    for (const item of feed.items) {
      // item: { title, link, contentSnippet, isoDate }
    }
  } catch (e) {
    // 单个源失败不影响其他源
  }
}
```

### 5.4 TwitterAPI.io — 社交媒体（预留）

接口预留，待获取 API Key 后接入。

```typescript
// 搜索推文
const res = await fetch('https://api.twitterapi.io/twitter/search', {
```

### 5.5 统一数据格式

所有数据源输出统一格式，方便后续 AI 处理和去重：

```typescript
interface RawArticle {
  source: 'firecrawl' | 'hackernews' | 'rss' | 'twitter';
  title: string;
  url: string;
  snippet: string;       // 内容摘要/片段
  publishedAt: string | null;
  metadata?: Record<string, unknown>;  // 来源特有数据
}
```

去重策略：以 URL 为唯一键，已存在于数据库的 URL 跳过。

### 6.2 AI 提示词设计

**内容验证提示词**：

```
你是一个技术热点分析专家。请分析以下文章是否为真实的技术热点。

文章标题：{title}
文章来源：{source}
文章摘要：{snippet}
监控关键词：{keyword}

请以 JSON 格式返回分析结果：
{
  "verified": true/false,       // 是否为真实热点（非标题党、非伪冒）
  "score": 0-100,               // 与关键词的相关性评分
  "reason": "判断理由",
  "summary": "50字以内的中文摘要"
}

判断标准：
1. 内容是否来自可靠来源
2. 标题是否与实际内容一致（非标题党）
3. 是否为伪冒/虚假消息
4. 与监控关键词的相关程度
```

**趋势分析提示词**：

```
你是一个技术趋势分析专家。请分析以下文章列表，归纳出当前的技术热点趋势。

文章列表：
{articles_json}

请以 JSON 格式返回，每个趋势包含：
{
  "trends": [
    {
      "category": "分类（如：AI大模型、编程语言、开源项目）",
      "title": "趋势标题",
      "summary": "100字以内的趋势描述",
      "heat_level": 1-5,
      "article_urls": ["相关文章URL"]
    }
  ]
}
```

### 6.3 模型选择策略

| 场景 | 推荐模型 | 说明 |
|------|----------|------|
| 免费使用 | `google/gemini-2.0-flash-exp:free` | 50 请求/天免费 |
| 有额度 | `openai/gpt-4.1-mini` | 更精准，成本低 |
| 批量分析 | `meta-llama/llama-4-maverick:free` | 上下文长，适合批量 |

### 6.4 优化策略

- **预过滤**：先用关键词匹配标题，只有命中的才送 AI 分析（节省配额）
- **批量请求**：多篇文章合并成一个请求，减少 API 调用次数
- **结果缓存**：同一 URL 不重复分析（数据库去重）
- **降级处理**：AI 不可用时，仅用关键词匹配，不做验证

## 七、通知机制

### 7.1 WebSocket 实时推送

```typescript
// 后端：使用 ws 库
import { WebSocketServer } from 'ws';

const wss = new WebSocketServer({ server: httpServer, path: '/ws' });
const clients = new Set<WebSocket>();

wss.on('connection', (ws) => {
  clients.add(ws);
  ws.on('close', () => clients.delete(ws));
});

// 广播新热点
function broadcast(event: { type: string; data: any }) {
  const msg = JSON.stringify(event);
  clients.forEach(ws => {
    if (ws.readyState === ws.OPEN) ws.send(msg);
  });
}
```

```typescript
// 前端：自定义 hook
function useWebSocket(url: string) {
  const [messages, setMessages] = useState<any[]>([]);

  useEffect(() => {
    const ws = new WebSocket(url);
    ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      setMessages(prev => [data, ...prev]);

      // 浏览器通知
      if (data.type === 'hotspot' && Notification.permission === 'granted') {
        new Notification('🔥 新热点发现！', {
          body: data.data.title,
          icon: '/radar-icon.png'
        });
      }
    };
    ws.onclose = () => setTimeout(() => /* 重连逻辑 */, 3000);
    return () => ws.close();
  }, [url]);

  return messages;
}
```
